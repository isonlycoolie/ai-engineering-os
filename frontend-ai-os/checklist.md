# Checklist

## Before opening a PR

- [ ] Acceptance criteria mapped to visible UI behavior
- [ ] Loading, error, empty, and success states implemented
- [ ] Forms validate and surface server errors correctly
- [ ] Keyboard navigation and labels verified
- [ ] No secrets or internal stack traces exposed to users
- [ ] Tests updated or added for critical paths
- [ ] Lint and type checks pass
- [ ] Human reviewer assigned
